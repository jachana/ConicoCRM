Conico PMS

Guía de inicio rápido · Sistema de gestión para Distribuidora Conico Ltda.

Desarrollo local

1

Requisitos

Docker Desktop y Git instalados.

2

Clonar y configurar

git clone git@github.com:jachana/ConicoCRM.git
cd ConicoCRM
cp .env.example .env   # editar con tus valores
3

Variables de entorno mínimas .env

DATABASE_URL=postgresql://conico:conico@db:5432/conico
SECRET_KEY=cambia-esto-por-un-secreto-largo
CORS_ORIGINS=http://localhost:5173
4

Levantar servicios

docker compose up --build
ServicioURL
Frontendhttp://localhost:5173
Backend APIhttp://localhost:8000
Swagger docshttp://localhost:8000/docs
5

Crear usuario admin inicial

docker compose exec backend python scripts/seed_admin.py
Crea admin@conico.cl / admin123. Cambiar después del primer login.

Producción — Coolify

Variables de entorno requeridas

VariableDescripción
DATABASE_URLpostgresql://conico:<password>@db:5432/conico
SECRET_KEYString aleatorio largo (mín. 32 chars)
CORS_ORIGINSURL del dominio, ej. https://crm.conico.cl
POSTGRES_PASSWORDContraseña de la base de datos

Variables opcionales (email)

VariableDescripción
SMTP_HOSTServidor SMTP
SMTP_PORTPuerto (default: 587)
SMTP_USERUsuario SMTP
SMTP_PASSWORDContraseña SMTP
SMTP_FROMDirección remitente

Pasos en Coolify

1

Nuevo recurso → Docker Compose

2

Repositorio: git@github.com:jachana/ConicoCRM.git

3

Archivo compose: docker-compose.prod.yml

4

Agregar variables de entorno → Deploy.

5

Crear admin inicial (una sola vez):

docker compose -f docker-compose.prod.yml exec backend python scripts/seed_admin.py
Las migraciones de base de datos corren automáticamente en cada deploy antes de iniciar el backend.
