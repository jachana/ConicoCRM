4. Roadmap propuesto (priorizado)

Wave 1 — Hardening producción (2 sprints)

Antes de vender: lo que hace al sistema confiable para una Pyme operando con dinero real.

  1. P0 Audit log global (modelo + middleware + vista admin filtrable)
  2. P0 Backups Postgres automáticos (cron + S3/Glacier) + script de restore probado
  3. P0 CI: lint TS + pytest backend + vitest frontend + build Docker en cada PR
  4. P0 Boleta electrónica 39/41 (vía Lioren) + flujo desde NV "boleta o factura"
  5. P0 Guía de despacho electrónica 52 desde NV
  6. P0 Observabilidad: logs estructurados (loguru/JSON), Sentry, /healthz
  7. P1 2FA (TOTP) en login; reset password por email

Wave 2 — CRM diferenciador (3 sprints)

  1. P0 Pipeline / Oportunidades (kanban + conversión a cotización)
  2. P0 Timeline unificado por cliente/empresa (cotis + NV + facturas + tareas + notas + llamadas)
  3. P1 Notas/llamadas registradas en cliente y empresa
  4. P1 Notificaciones (campana in-app + email digest configurable)
  5. P1 Plantillas de email configurables (cotización, factura, recordatorio)
  6. P2 Importación masiva clientes/empresas via Excel
  7. P2 Bulk actions en tablas (cambiar estado, exportar selección, eliminar lote)

Wave 3 — Tributario completo (2 sprints)

  1. P0 Recepción / intercambio DTE de proveedores + acuse de recibo (RCV)
  2. P0 Libro de compras / ventas electrónico (export SII)
  3. P1 DTE 34 (factura exenta) + DTE 46 (factura de compra)
  4. P1 Propuesta F29 mensual

Wave 4 — Finanzas (3 sprints)

  1. P0 Conciliación bancaria (importar cartola Banco de Chile/BCI/Santander → match con pagos)
  2. P0 Comisiones por vendedor (config % + reporte mensual)
  3. P1 Multi-moneda + UF (TC manual o vía indicadores SII)
  4. P1 Exportación a contabilidad (Defontana / Softland / ContaNet) — al menos un conector
  5. P2 Cash flow + estado de resultados básico
  6. P2 Facturación recurrente / suscripciones

Wave 5 — Operación avanzada (2 sprints)

  1. P1 Multi-bodega (movimientos por bodega, transferencias)
  2. P1 Trazabilidad lote/serie en NV y OC (consolidar Lotes existentes)
  3. P2 POS minimal (boleta rápida, lectura código de barras, vuelto)

Wave 6 — Producto SaaS (4 sprints, opcional)

  1. P0 Multi-tenant: tenant_id en todas las tablas + middleware de scope
  2. P0 Customer portal (login cliente externo, ver sus facturas/cotizaciones, descargar PDF)
  3. P1 Mobile/PWA responsive para todas las páginas core
  4. P1 Integraciones medios de pago (Transbank Webpay, Mercado Pago, Khipu)
  5. P2 WhatsApp Business para envío de PDF y recordatorios
  6. P2 SSO Google/Microsoft
  7. P2 API pública + webhooks salientes para integradores

5. KPIs a instrumentar

  • Adopción interna: usuarios activos diarios, % vendedores con cotización abierta esta semana.
  • Velocidad de venta: días promedio cotización → NV, NV → factura, factura → pago.
  • Salud cobranza: facturas vencidas / emitidas, días promedio mora, ratio recuperación 30/60/90 días.
  • Calidad inventario: % productos con costo stale, ajustes manuales por mes, stock crítico vs alertas resueltas.
  • Confiabilidad SII: % DTE aceptado en primer intento, tiempo medio emitida → aceptada, rechazos por causa.
  • Plataforma: error rate API, p95 latencia endpoints calientes (search, sugerencias, dashboard), uptime.

6. Decisiones bloqueantes pendientes

  1. Stock NV: ¿descuento al crear o al despachar? — pendiente confirmar con cliente desde 2026-04-22.
  2. Pipeline: ¿estados fijos hardcoded o configurables por admin?
  3. Multi-tenant: ¿se vende como SaaS multi-cliente o queda single-tenant on-prem? Impacta Wave 6 y refactor de queries.
  4. POS: ¿Conico debe reemplazar caja o convivir con un POS externo (vía DTE 39)?
  5. Contabilidad: ¿integramos con un proveedor (cuál priorizar) o exportamos archivo plano?
